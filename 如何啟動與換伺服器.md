# AutoHelp 4.0 — 啟動與換伺服器說明

## 平常怎麼用（本機）

- **開機登入後會自動啟動**後端(3001)+前端(5173),不用做任何事。
  - 背後是工作排程任務「AutoHelp 自動啟動」在登入時跑 `start-autohelp.ps1`。
- 想手動啟動 / 確認有沒有在跑 → **雙擊桌面「啟動 AutoHelp」捷徑**。
  - 已經在跑就會顯示 `[OK]`,不會開兩份(重複點安全)。
- 開始生成:打開 <http://localhost:5173> → Figma 載入外掛 → Start Polling。
- 啟動紀錄在 `autohelp-start.log`(出問題時看這個)。

---

## 未來要換到別台伺服器（例如公司內網）

整套已經設計成「改設定就好,不用改程式」。換伺服器時改這 **3 個地方**:

### 1. 啟動腳本 `start-autohelp.ps1` 頂部
把這一行的網址換成新伺服器:
```powershell
$BackendUrl = "http://localhost:3001"
# 例:公司內網 => "http://192.168.1.50:3001"
```
(這會讓本機前端預設指向新後端)

### 2. 網頁 ⚙ Settings
打開網頁 → 右上 ⚙ Settings → **Backend URL** 填新伺服器網址、**API Key** 維持 `dev-key-12345`(或新伺服器設定的金鑰)。存在瀏覽器,改一次即可。

### 3. Figma 外掛輸入框
外掛 UI 的 **Backend URL** 欄填新伺服器網址即可。
- Figma 網域白名單已預先放寬成 `"*"`(任何網址都允許),所以**換內網 IP 不必重新匯入外掛**。
- 設定位置:`figma-bridge/manifest.json` 的 `devAllowedDomains`。

### 後端那台本身
在新伺服器的 `backend/.env` 設:
- `ANTHROPIC_API_KEY=`(你的金鑰)
- `PUBLIC_BASE_URL=`(這台對外的網址,例如 `http://192.168.1.50:3001`)— 決定它吐出的圖片網址。
- 上傳的符號圖在後端檔案系統,搬機器時記得一起搬 `backend` 的資料夾。

> 確切內網地址確定後,把它告訴我,我可以幫你一次把以上預設值改好。
